### 운영 환경

```
◦ 백엔드: Node.js(Express), PostgreSQL(Supabase), JWT 기반 인증
◦ 프론트엔드: React+Vite, Morpheus
◦ 개발 도구: VS Code, Android Studio, Postman
```

### 추진 배경

```
◦ 자기주도 학습 지속 한계와 고립된 환경
  - 서울대 신종호 교수 "목적의식 부재, 흥미 결여, 혼자 공부하기의 난이도" 지적
  - 수험 생활의 가장 큰 적은 고립감과 막연함이고, 이로 인해 중도 포기가 많음
◦ 기존 학습 도구의 한계
  - 기존의 열품타, Flipd 같은 앱들은 앉아있는 시간만 측정하기 때문에 실질적 학습량 보장 불가
  - 공부는 구체적인 단위로 수행되지만 기존 앱은 총 시간으로만 퉁쳐서 기록
  - 단순 시간 랭킹외에는 구체적인 학습 인증과 상호작용(피드백) 부족
```

### 프로젝트 결과

```
◦ 웹 서비스 프로토타입 구축
  - 챌린지 생성·참여부터 인증 업로드, 좋아요·응원 토글, 인증 횟수와 목표 빈도를 기반으로 그래프·GitHub 잔디형 캘린더 시각화 기능까지 하루 1회 인증 제한을 포함한 Study Mate 웹 서비스 프로토타입 구축
◦ 실시간 지표 갱신
  - Supabase + Express API 구현으로 실시간 지표(좋아요·응원·팔로우·참여자·인증글 수) 갱신
◦ AI 기반 안전·추천 기능
  - Gemini API로 욕설·비방 등 유해 표현 사전 차단과 인증글 태그 추출 및 누적하여 챌린지 추천 신뢰도를 높이고 챌린지 참여 진입장벽을 낮춤
```

### 주요 기능 설명

```
◦ 학습 인증
  - 인증글을 하루 1회 업로드하고 서버 검증으로 미참가·중복 업로드를 차단
◦ 달성률 시각화 및 GitHub 잔디형 캘린더
  - 주간·전체·최근 30일 달성률 그래프와 일별 누적 인증 횟수를 보여주는 GitHub 잔디형 캘린더로 시각화
◦ 소셜 상호작용 및 달성률 랭킹
  - 챌린지 좋아요, 인증글 응원, 친구 팔로우로 동기 부여를 강화하고, 최근 30일 달성률 기반 팔로워·팔로잉 간 랭킹을 실시간 갱신(+1/-1 변동 표기)
◦ AI 욕설·비방 검증
  - 인증글 업로드 시 Gemini API로 욕설·비방 등 유해 표현을 사전 차단
◦ AI 추천 챌린지
  - 인증글 업로드 시 Gemini API로 인증글을 분석하여 관련 태그 추출 후 사용자별로 점수처럼 누적시켜 추천 목록 기능을 이용할 때 사용자의 태그와 챌린지의 태그가 많이 겹칠수록 상위에 노출
```

### 문제점 및 향후 개선 방향

```
◦ AI 인증 검증 도입
  - AI 욕설·비방 검증을 넘어 챌린지 적합성·허위 내용까지 자동 검증하는 피드백 절차 마련
◦ 오프라인 커뮤니티 연계 확장
  - 학교, 학원뿐만 아니라 독서실, 스터디 카페, 동호회 등 다양한 오프라인 그룹과 연계된 전용 챌린지 및 랭킹 도입
◦ 사용자 인증글에 기반한 학습 리포트 생성
  - 누적 인증 데이터를 기반으로 학습 리포트를 생성하여 사용자에게 객관적인 학습 기록을 제공하고, 사용자가 학습 방향 설계에 참고
```

### 활용방안 및 기대효과

```
◦ 학습 루틴 고착화
  - 막연하게 시간만 채우는 보여주기식 공부에서 벗어나서 구체적인 과제를 완수하고 인증하는 밀도 높은 학습 습관을 완성합니다.
◦ 사회적 동기 부여
  - 친구들의 실시간 인증과 랭킹이 페이스메이커 역할을 하면서 혼자 공부할 때 겪는 고립감과 작심삼일의 마음을 극복하고 꾸준함을 유지하게 됩니다.
◦ 자기 인식 향상
  - 직관적인 그래프와 캘린더로 자신의 학습 패턴을 객관적으로 파악하고 성취감을 높이며 부족한 점을 스스로 보완하는 자기주도력을 기릅니다.
```
