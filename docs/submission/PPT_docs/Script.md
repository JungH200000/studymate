## 2 슬라이드 : 목차

발표는 보시는 목차 순서대로 진행하겠습니다. 프로젝트 개요를 시작으로, 추진 체계, 시스템 개발 내용, 그리고 시연 및 기대효과 순으로 말씀드리겠습니다.
먼저 프로젝트 개요에서 추진 배경과 타깃, 개발 환경을 소개합니다.
이어서 추진체계와 일정에서는 팀 역할 분담과 개발 일정을 보여드리겠습니다.
세 번째로 시스템 개발 내용에서는 전체 구성도와 UI를 보고, 챌린지 탐색 및 참여 → 학습 인증과 AI 검증 → AI 추천 → 성과 확인 및 경쟁 순으로 설명드리겠습니다.
마지막으로 시연 영상 흐름, 기대효과, 향후 계획을 말씀드리며 마무리하겠습니다.

## 3 슬라이드 : 프로젝트 개요-추진 배경

먼저 추진 배경입니다.
왜 혼자 하는 공부는 어려울까요?
서울대 신종호 교수는 그 이유로 목적의식 부재와 흥미 결여를 꼽았습니다. 특히 수험 생활의 가장 큰 적은 '고립감'과 '막연함'입니다. 혼자 공부하다 보면 내가 잘하고 있는지 불안해지고, 이로 인해 중도 포기하는 경우가 많습니다.
하지만 기존의 열품타·Flipd(플립) 같은 앱들은 단순히 앉아 있는 시간만 측정하기 때문에 진짜 학습량을 확인하기 어렵고 상호작용도 부족해 동기부여가 약합니다.
그래서 저희는 '시간을 얼마나 보냈는지'가 아니라 '오늘 어떤 과제를 했는지'를 하루 한 번 인증하고 친구들과 함께 소통하며 공부할 수 있는 플랫폼을 설계했습니다.

## 4 슬라이드 : 프로젝트 개요-활용 목적 및 타겟 오디언스

핵심 한 줄은 “Time → Task, 앉아있는 시간이 아니라 끝낸 공부를 관리합니다.”
타겟은 공시생·취준생·중고생 등 모든 자기주도 학습자입니다.

- 성취감: 시간 기록이 아닌 구체적 학습 목표 인증으로 실질 성취를 제공합니다.
- 연대감: 친구가 지켜보는 페이스메이커로 고립감을 줄입니다.
- 시각화: 달성률 그래프와 인증 기록으로 성장 경로를 눈으로 확인합니다.

## 5 슬라이드 : 프로젝트 개요-프로젝트 개발 환경

개발 환경은 그림과 같이 구성했습니다.
프론트엔드는 React·Vite 위에 Morpheus를 얹어 모바일 앱을 만들었고,
서버는 Node.js Express에 JWT 인증과 쿠키 기반 Refresh 토큰 구조를 사용했습니다.
데이터베이스는 Supabase PostgreSQL을 기반으로 하고,
AI 태그 추출 워커도 Supabase에 연결해 운영했습니다.
개발과 테스트는 VS Code, Android Studio, Postman, GitHub로 진행했습니다.

김지훈 팀원은 홈·인증·통계 UI 등 핵심 프론트 화면을 구현하고 시연 영상 촬영도 맡았습니다.

## 7 슬라이드 : 추진체계 및 일정-개발 일정

개발 일정은 총 6주로 구성했습니다.

셋째 주부터 다섯째 주까지 프론트엔드 구조와 UI, 핵심 API를 집중 구현했습니다.
마지막 주에는 최종 테스트와 시연 영상, 발표 자료를 준비해 마무리했습니다.

## 8 슬라이드 : 시스템 개발 내용-전체 시스템 구성도

전체 시스템 흐름을 한 장으로 정리했습니다.
저희 시스템은 JWT 기반 REST 아키텍처와 Gemini 연동 데이터 흐름을 중심으로 설계되었습니다.
왼쪽 클라이언트는 React·Vite와 Morpheus를 결합해 만든 앱이고, JWT로 인증된 API 요청을 중앙의 Node.js Express 서버에 보냅니다.
서버는 이 요청을 받아 Supabase DB와 통신하여 데이터를 저장하거나 조회합니다. 이 과정에서 Gemini API를 호출해 욕설을 필터링하거나 태그를 추출하고, 최종 결과는 JSON 형태로 클라이언트에 반환되어 실시간으로 화면을 갱신합니다.

---

## 9 슬라이드 : 시스템 개발 내용-서비스 주요 기능

저희 서비스를 세 단계로 정리해서 말씀드리겠습니다.
첫째, 챌린지 탐색과 참여입니다. 최신/추천 탭에서 목록을 보고, 마음에 들면 버튼 하나로 참여·취소와 좋아요를 눌러 바로 지표가 바뀝니다.
둘째, 학습 인증과 AI 검증입니다. 하루 한 번 JSON으로 목표·요약·자료·시간을 적어 올리고, 업로드 직전에 Gemini가 욕설·비방을 검사해 안전하지 않으면 차단합니다.
셋째, 성과 확인과 경쟁입니다. 주간·전체·최근 30일 달성률과 잔디형 캘린더로 패턴을 시각화하고, 팔로워/팔로잉 네트워크의 실시간 랭킹으로 서로 자극을 주고받습니다.

## 10 슬라이드 : 시스템 개발 내용-서비스별 설명-챌린지 탐색 및 참여

첫 번째 기능 챌린지 탐색과 참여에 대해 말씀드리겠습니다.
왼쪽처럼 최신/추천 목록에서 참여 버튼을 누르면 참여 상태와 숫자가 즉시 바뀌고, 좋아요도 같은 방식으로 토글됩니다.
가운데처럼 챌린지를 새로 만들 때는 제목, 기간, 빈도(주/일), 태그만 입력하면 바로 목록에 반영됩니다.
오른쪽처럼 좋아요·참가자 수와 참가자 목록을 보며 누가 함께하는지 확인해 연대감을 높일 수 있습니다.

---

## 11 슬라이드 : 시스템 개발 내용-서비스별 설명-학습 인증 및 AI 검증

두 번째 기능 학습 인증과 AI 검증, 분석에 대해 말씀드리겠습니다.
사용자가 목표·요약·시간·자료를 JSON으로 채워 하루 공부를 정리하면, 서버가 먼저 “이 사용자가 참여자인지, 오늘 이미 올렸는지”를 KST 기준으로 확인합니다.
통과하면 업로드 직전에 Gemini가 욕설·비방 여부를 검사해 안전하지 않으면 거부합니다.
올라간 인증글은 목록에 바로 보이고, 친구들이 응원 토글을 눌러 상호작용하며 동기부여를 주고받습니다.

---

## 12 슬라이드 : 시스템 개발 내용-서비스별 설명-AI 추천 챌린지

이렇게 올린 인증글을 바탕으로 AI 추천 챌린지 기능이 동작합니다.
인증글이 올라오면 Gemini가 최대 5개의 태그를 뽑아 user_tags 테이블에 가중치로 누적합니다.
홈의 추천 탭에서는 내 가중치와 챌린지 태그를 매칭해 점수 높은 순으로 정렬하고, 데이터가 없을 때는 최신순으로 보여 빈 화면이 없도록 했습니다.

---

## 13 슬라이드 : 시스템 개발 내용-서비스별 설명-성과 확인 및 경쟁

세 번째 기능 성과와 경쟁 파트입니다.
잔디형 캘린더가 일별 인증 빈도를 색으로 보여 꾸준함을 자극하고, 주간·전체·최근 30일 달성률은 인증 직후 그래프와 파이 차트에 반영돼 오늘까지 얼마나 했는지 바로 확인할 수 있습니다.
랭킹은 최근 30일 달성률로 나와 팔로워·팔로잉 순위를 계산해 보여주고, 등락 폭을 같이 표시해 응원과 경쟁을 동시에 유도합니다.

## 14 슬라이드 : 시연 영상 및 기대 효과-시연 영상

지금부터 시연 영상을 재생하겠습니다.
영상에서는 챌린지 등록 → 챌린지 목록 → 참여 버튼 → 좋아요 버튼 → 특정 챌린지 진입 → 인증글 등록(욕설 필터 확인) → 인증글 목록 → 응원 버튼 → 사용자 페이지(참여·등록 챌린지) → 달성률 그래프 순으로 보시겠습니다.

## 15 슬라이드 : 시연 영상 및 기대효과-프로젝트 기대효과

해당 프로젝트를 통해 기대되는 효과는 세 가지입니다.

첫째, 결과 중심 습관 고정입니다. 하루 1회 인증과 정형 기록으로 보여주기식 시간 채우기에서 “과제 완료” 습관으로 전환합니다.
둘째, 안전한 연대·지속 동기입니다. Gemini가 유해 표현을 차단하고 응원·랭킹이 페이스메이커 역할을 해 고립감을 줄이며 동기를 유지합니다.
셋째, 데이터 기반 자기주도력 강화입니다. 달성률 그래프와 잔디 캘린더, 태그 기반 추천으로 자신의 패턴을 파악하고 학습 방향을 설계합니다.

---

## 16 슬라이드 : 시연 영상 및 기대효과-향후 계획 및 보완사항

향후 계획은 네 가지입니다.

첫째, 푸시 알림과 학습 타이머를 연동해 미인증자에게 '재촉하기' 알림을 보내고, 타이머 종료와 동시에 인증을 제안해 기록 정확도를 높이겠습니다.
둘째, AI 인증 검증을 고도화해 욕설 차단을 넘어서 챌린지 적합성·허위 여부·간단한 피드백까지 자동 판정하도록 할 예정입니다.
셋째, 확장성을 위해 독서실, 스터디 카페, 동호회 등 오프라인 커뮤니티와 연계된 전용 챌린지를 도입하겠습니다.
넷째, 누적 인증 데이터를 기반으로 주간/월간 학습 리포트를 자동 생성해 사용자가 학습 방향 설계에 활용할 수 있게 하겠습니다.
이상으로 발표를 마치겠습니다. 감사합니다.
