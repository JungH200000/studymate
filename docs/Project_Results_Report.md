### 운영 환경

```
◦ 백엔드: Node.js(Express), PostgreSQL(Supabase), JWT 기반 인증
◦ 프론트엔드: React+Vite, Morpheus
◦ 개발 도구: VS Code, Android Studio, Postman
```

### 추진 배경

```
◦ 자기주도 학습의 난관
  - 목적의식 부재, 흥미 결여, 혼자 공부하기의 높은 난이도로 인해 혼자 공부 지속이 어려움
◦ 주의력 지원 필요성 증가
  - ADHD 진료 학생 수가 2018년 3만 8천명에서 2022년 6만 4천명으로 약 2배 증가
◦ 디지털 환경의 역설
  - 청소년 스마트폰 과의존 위험군 비율 42.6%, 주의 분산 환경 확대
◦ 기존 도구의 한계
  - SNS는 학습 목표 설계 부족, 습관앱은 동료 상호작용 약함
```

### 프로젝트 결과

```
◦ 웹 서비스 프로토타입 구축
  - 하루 1회 인증 제한과 응원·달성률 피드백이 작동하는 Study Mate 웹 서비스 프로토타입 구축
◦ 학습 루틴 전과정 통합
  - 챌린지 생성/참여, 인증 업로드, 좋아요·응원 토글, 달성률 시각화 기능으로 핵심 학습 루틴 전 과정을 통합
◦ 실시간 지표 갱신
  - Supabase 기반 서버·DB 구조 설계 및 Express API 구현으로 실시간 지표(좋아요·응원·팔로우·참여자·인증글 수) 갱신
◦ 탭형 챌린지 상세/인증 뷰
  - 챌린지 카드를 누르면 상세정보·인증글 탭으로 분리된 화면에서 기간·빈도·좋아요/참여 지표와 인증 피드를 한 번에 확인
```

### 주요 기능 설명

```
◦ 학습 인증
  - 챌린지 상세 화면에서 상세정보/인증글 탭을 전환하며 기간·빈도·좋아요/참여 지표와 인증 피드를 동시에 열람
  - 목표·요약·학습 시간·학습 자료를 담은 JSON 기반 인증글을 하루 1회 업로드하고 서버 검증으로 미참가·중복 업로드를 차단
  - 인증글 목록에서 최신 인증글과 응원 토글·응원자 목록·내 응원 여부를 함께 확인, 업로드 직후 주간·전체 실적이 즉시 갱신
◦ 달성률 시각화 및 GitHub 잔디형 캘린더
  - 사용자 페이지의 참여/등록 챌린지 카드에서 주간 달성률 막대 그래프를 확인하고, 달성률을 탭하면 전체·최근 30일 달성률 그래프와 일별 누적 인증 횟수를 보여주는 GitHub 잔디형 캘린더 페이지로 이동
  - 인증 횟수와 목표 빈도를 기반으로 각 그래프·캘린더를 자동 계산해 누적 학습 패턴을 시각적으로 제공
◦ 소셜 상호작용
  - 챌린지 좋아요, 인증글 응원, 친구 팔로우로 동기 부여 강화
```

### 개발 특이 사항

```
◦ 더미 데이터 생성에 AI 활용
  - 챌린지 목록과 인증글 목록, 달성률 시각화 기능 구현을 위해 AI를 이용해 더미 데이터를 생성
```

### 문제점 및 향후 개선 방향

```
◦ 푸시 알림·학습 타이머 연동
  - 미인증 참여자 대상 ‘재촉하기’ 알림을 도입하여 챌린지 동기 유지
  - 학습 타이머·인증 연동으로 기록 정확도 향상
◦ AI 인증 검증 도입
  - AI 인증 검증을 도입하여 작성한 인증글이 챌린지와 적합한지 등 피드백 절차 마련
◦ 학교별 챌린지 개설하여 지역별/학교별 랭킹으로 확장
  - 학교 단위 챌린지 개설 및 지역별/학교별 랭킹 시스템으로 확장하여 경쟁과 참여 유도
◦ 사용자 인증글에 기반한 학습 리포트 생성
  - 누적 인증 데이터를 기반으로 학습 리포트를 생성하여 사용자에게 객관적인 학습 기록을 제공하고, 사용자가 학습 방향 설계에 참고
```

### 활용방안 및 기대효과

```
◦ 학습 루틴을 꾸준히 유지
  - 응원 토글과 달성률 그래프 시각화로 꾸준한 학습을 유지할 수 있다.
◦ 자기주도 학습 리듬 형성
  - 하루 1회 인증으로 목표→학습→인증 과정을 고정
◦ 주의력·디지털 과의존 문제 대응
  - ADHD 진료 학생 수가 최근 5년간 약 2배 증가하고, 스마트폰 과의존 위험군이 42.6%에 달하는 환경에서 집중형 학습 SNS로 대안 제공
```
