## 8 슬라이드 : 시스템 개발 내용-전체 시스템 구성도(사이트 맵)

### 작성 사항

- 본 프로젝트를 한 눈에 이해할 수 있는 전체 시스템 구성도 삽입 및 설명
- 꼭 사용자/관리자로 나누어 설명할 필요 없음

### 예상 슬라이드 내용

← → ↑ ↓

```
# 챌린지 등록
[프론트엔드] 챌린지 등록 요청 → [서버] 챌린지 등록 API (검증·권한 확인) → [DB]챌린지 저장 후 결과 응답

# 챌린지 목록
[프론트엔드] 홈 화면 진입 시 챌린지 목록 요청 ← [서버] 챌린지 조회 API (페이징·정렬) ← [DB] 챌린지 데이터 조회 후 응답

# 챌린지별 좋아요 기능
[프론트엔드] 챌린지 좋아요 요청 → [서버] 좋아요 API (중복 방지·카운트 갱신) → [DB] 좋아요 상태 저장 후 응답

# 챌린지별 인증글 등록
[프론트엔드] 인증글 등록 요청 → [서버] 인증글 등록 API (검증·권한 확인) → [DB] 인증글 저장 후 결과 응답

# 챌린지별 인증글 목록
[프론트엔드] 챌린지에 진입 시 인증글 목록 요청 → [서버] 인증글 조회 API (페이징·정렬) → [DB] 인증글 데이터 조회 후 응답

# 챌린지별 인증글 응원
[프론트엔드] 인증글 응원 요청 → [서버] 응원 API (중복 체크·카운트 갱신) → [DB] 응원 상태 저장 후 응답

# 사용자별 팔로우
[프론트엔드] 팔로우 요청 → [서버] 팔로우 API (상태 확인) → [DB] 팔로우 관계 저장 후 결과 응답

# 달성률 시각화
[프론트엔드] 주간/전체/최근30일 달성률 요청 → [서버] 달성률 API (집계) → [DB]달성률 데이터 계산·조회 후 응답
```

- 검증 : 필수 항목 확인, (인증글)하루 1회 제한 확인 등
- 권한 : 로그인한 사용자인지
